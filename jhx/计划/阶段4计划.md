# 阶段 4 计划：网站 MVP 收尾

> 2026-06-11 制定。对应总计划"阶段 4"。目标一句话：不加新功能，把 demo 从"功能都有"推到"可以长时间开着用"——状态可信、渲染跟手、不留垃圾文件。

## 1. 范围

**做（三件事 + 一项保障）：**

1. 健康状态周期轮询
2. 试妆渲染改 requestAnimationFrame
3. 临时文件清理（隐私收尾）
4. 完整闭环回归验收

**不做：**

- 语音输入（阶段 5）、face parsing ONNX / 商品库 / PixelFree 重做（阶段 6）、agent（阶段 7）。
- 不动视觉风格（杂志风刚定稿）。
- 不做 stream 自动重连（出错走现有草稿兜底已足够）。

## 2. 健康轮询

现状：`checkHealth()` 只在页面加载时跑一次，Ollama 中途掉线 UI 不会变化。

改动（app.js）：

- `checkHealth` 改为每 20 秒轮询一次（`setInterval`），加载时立即跑第一次。
- 状态变化才更新 DOM，避免每 20 秒闪一次。
- Ollama 掉线 → pill 变 "Ollama 未连接"（warn）；后端掉线 → "后端未连接"（bad）；恢复后自动变绿。
- 后端版本检测（features 缺失 → "后端版本旧"）逻辑保留不动。
- 轮询请求超时设短（5 秒，AbortController），避免挂起请求堆积。

验收：

- 手动结束 Ollama 进程，20 秒内 pill 变 warn；期间提问仍能得到草稿卡 + 本地规则建议，无报错弹窗。
- 重新启动 Ollama，20 秒内 pill 恢复绿色，流式恢复可用。

## 3. 试妆渲染改 rAF

现状：口红/腮红画在 overlay 上，由 220ms 定时器驱动，相对 30/60fps 视频肉眼可见地"漂"和抖。

改动（app.js）：

- 拆分两条节奏：
  - **分析循环**：保持 220ms `setInterval`，负责检测、特征、FaceState（不变）。
  - **渲染循环**：`requestAnimationFrame`，每帧用"最近一次分析结果"重画 overlay（试妆 + debug 覆盖层）。
- `analyzeFrame` 不再直接调 `renderOverlay`；渲染循环读取 `currentAnalysis`。
- 节流细节：
  - camera 模式且摄像头运行中才跑 rAF 循环；停止摄像头或切到 image 模式时停止循环（image 模式渲染一次静态结果即可）。
  - 试妆关闭且非 debug 时跳过重画，避免空转耗电。
- 滑杆调整（口红/腮红强度）即时生效：渲染循环每帧读 `makeupSettings()`，现有 input 监听只需触发一次分析。

验收：

- 开启试妆后缓慢转头，唇彩贴合嘴唇移动，无 220ms 台阶式跳动（肉眼对比改前后）。
- 任务管理器粗查：改动后 CPU 占用无明显上升（rAF 重画成本应低于检测成本）。
- `?debug=1` 的关键点覆盖层同样每帧跟随。

## 4. 临时文件清理

现状：PixelFree 快照把用户摄像头帧写进原生 demo 目录的 `IMG_2406.png`，用后不删，违反总计划 §8"默认不保存用户图片"。

改动（server.py）：

- 快照输入文件改名为专用文件（如 `snapshot_input.png`），不再覆盖 SDK 自带示例图（需同步确认原生 demo 读取的文件名；若 exe 写死 `IMG_2406.png` 则维持原名，仅做删除管理）。
- 启动原生进程后，后台延迟 ~10 秒删除输入文件（进程启动时已读入内存）；启动失败立即删除。
- 服务启动时清理上次会话遗留的快照输入文件。
- 抽出 `cleanup_snapshot_files(dir)` 纯函数供单测。

仓库整理（顺手，一次性）：

- 根目录 486KB 的 `pixelfree snapshot.png` 调试截图移入 `docs/`（或删除，问用户）。
- 确认 `.gitignore` 覆盖 `__pycache__`、快照输入文件。

验收：

- 触发快照后 ≤15 秒，输入图片从磁盘消失；原生窗口显示不受影响。
- 重启服务后无残留快照文件。
- 新增清理函数单测通过。

- [ ] 快照文件用后即删（第 4 节验收）。

## 6. 已知局限（写明，不算验收失败）

- 柔肤仍是全画面 CSS filter 占位，真正的皮肤区域磨皮等阶段 6 接 face parsing / 原生引擎。
- rAF 渲染用的是最近一次分析的关键点，快速甩头时唇彩最多滞后一个分析周期（~220ms），属可接受；插值平滑留作可选优化。
- 健康轮询只覆盖后端与 Ollama，不检测摄像头被其他应用抢占等设备级异常。
